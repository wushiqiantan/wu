# AI设计工作流平台 — 后端部署指南

> 后端代理方案：隐藏真实 API Key + 用量控制 + 权限校验

---

## 📁 后端文件结构

```
backend/
├── server.js          # 主服务器文件（Express + 代理 + 限流 + 鉴权）
├── package.json       # 依赖配置
├── .env               # 环境变量（API Key，**绝不提交到 Git**）
├── .env.example       # 环境变量模板
└── .gitignore         # 忽略 node_modules 和 .env

js/api-config.js       # 前端配置（已改为调用后端代理）
start.bat              # Windows 一键启动脚本
start.sh               # macOS/Linux 一键启动脚本
```

## 🚀 快速启动（3步）

### 前提条件
- [Node.js](https://nodejs.org/) 18+ 已安装
- 后端已配置好 `.env` 文件（API Key 已填入）

### 第一步：启动后端

**Windows：**
```bash
# 在项目根目录
双击 start.bat
# 或命令行
.\start.bat
```

**macOS/Linux：**
```bash
# 在项目根目录
chmod +x start.sh
./start.sh
```

**首次启动会自动安装依赖（npm install），之后直接启动。**

如果看到如下输出，说明后端已启动：
```
╔══════════════════════════════════════════════════════╗
║  AI景观设计工作流平台 - 后端服务器                   ║
╠══════════════════════════════════════════════════════╣
║  端口: 3000                                          ║
║  地址: http://localhost:3000                         ║
╠══════════════════════════════════════════════════════╣
║  已配置AI服务: DeepSeek, Kimi, OpenAI                ║
╚══════════════════════════════════════════════════════╝
```

### 第二步：打开前端

后端启动后，浏览器访问：
```
http://localhost:3000
```

后端会自动服务前端静态文件（`index.html`），**无需再单独打开 HTML 文件。**

### 第三步：验证连通

按 `F12` 打开浏览器控制台，运行：
```javascript

```

应返回类似：
```json

  "services": {
```

## 🔐 安全机制

### 1. API Key 隐藏
- 三个 AI Key 只存在于后端 `.env` 文件
- 前端 `js/api-config.js` 中只有后端地址和代理 Token
- 前端代码中**没有任何真实 API Key**

### 2. 权限校验（Bearer Token）
每次调用 AI 接口都需要：
```http
Authorization: Bearer ai-design-backend-2024-secret-key
```

如果 Token 错误，返回：
```json
{ "error": "无效的 API Token", "code": "INVALID_TOKEN" }
```

### 3. 用量控制

| 限制类型 | 默认值 | 说明 |
|---------|--------|------|
| 每日总请求 | 200 次 | 所有文本 AI 调用合计 |
| 每日图像生成 | 50 次 | GPT Image2 调用 |

达到上限后返回：
```json
{ "error": "今日 API 调用次数已达上限 (200次)", "code": "DAILY_LIMIT_EXCEEDED" }
```

### 4. 请求日志
后端记录每次请求：
- 时间戳
- 请求类型（chat / image）
- 客户端 IP
- 成功/失败状态

## 📡 API 端点

| 端点 | 方法 | 说明 | 鉴权 |
|------|------|------|------|
| `/api/health` | GET | 健康检查 + 服务配置 | 无需 |
| `/api/config` | GET | 前端配置（无 Key） | 无需 |
| `/api/ai/chat` | POST | 文本 AI 代理（DeepSeek/Kimi） | 需要 |
| `/api/ai/image` | POST | 图像生成代理（GPT Image2） | 需要 |
| `/api/usage` | GET | 用量统计 | 需要 |

### 请求示例

**文本生成（DeepSeek）：**
```bash
curl -X POST http://localhost:3000/api/ai/chat \
  -H "Authorization: Bearer ai-design-backend-2024-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"service":"deepseek","messages":[{"role":"user","content":"Hello"}]}'
```

**图像生成：**
```bash
curl -X POST http://localhost:3000/api/ai/image \
  -H "Authorization: Bearer ai-design-backend-2024-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"prompt":"现代城市公园鸟瞰图","n":4,"size":"1024x1024","quality":"medium"}'
```

## ⚙️ 配置说明

### 修改后端 Token（必须！）

打开 `backend/.env`：
```
API_SECRET=ai-design-backend-2024-secret-key
```

**改成你自己的复杂字符串**，然后同步修改前端 `js/api-config.js`：
```javascript
backend: {
    apiSecret: '你改的新字符串'
}
```

### 修改用量限制

编辑 `backend/.env`：
```
MAX_DAILY_REQUESTS=500
MAX_IMAGE_REQUESTS_PER_DAY=100
```

重启后端生效。

### 修改后端端口

编辑 `backend/.env`：
```
PORT=3001
```

同步修改前端 `js/api-config.js`：
```javascript
baseURL: 'http://localhost:3001'
```

## 🔄 开发模式

如果需要前后端分离开发：

```bash
# 终端 1：启动后端（自动服务前端）
.\start.bat

# 终端 2：或单独修改后端代码
node backend/server.js
```

修改 `backend/server.js` 后自动生效（nodemon 模式）：
```bash
cd backend
npm run dev
```

## 📦 生产部署

本后端是轻量级 Express 应用，可部署到任意 Node.js 环境：

### 部署清单
1. 上传 `backend/` 目录和 `index.html` + `js/` + `css/`
2. 服务器上创建 `.env` 并填入真实 API Key
3. 服务器上运行 `npm install` 安装依赖
4. 使用 `pm2` 或 `systemd` 保持进程常驻

### 示例（PM2）
```bash
npm install -g pm2
pm2 start backend/server.js --name "ai-design"
pm2 save
pm2 startup
```
