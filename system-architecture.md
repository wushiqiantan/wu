# AI设计工作流平台 — 系统架构与实现路线图

> 定位：面向设计师、学生、设计教师的个人轻量化AI设计平台
> 核心：节点化工作流 × 多AI协同 × 人机确认机制 × 订阅制

---

## 一、核心工作流全景（节点化）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         【项目启动】                                          │
│    改造项目 ──→ 选择项目类型 ──→ 加载对应模板库                               │
│    新建项目 ──→ (城市公园/社区景观/校园改造/商业空间/居住区...)                  │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【需求分析与文本生成】DeepSeek                            │
│    用户输入需求 ──→ DeepSeek分析 ──→ 生成：                                   │
│     • 设计策略文本                                                            │
│     • 设计理念阐述                                                            │
│     • 分区功能说明                                                            │
│     • 文本讲稿（汇报用）                                                       │
│     • 图像提示词大纲（给GPT Image2的输入）                                      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【图像提示词模板系统】                                    │
│    按项目类型内置专业提示词模板：                                               │
│     • 鸟瞰图 / 人视效果图 / 剖面图 / 节点详图 / 夜景图 / 季节变化图             │
│     • 植物配置图 / 材质铺装图 / 灯光分析图                                     │
│    用户可基于模板调优，或完全自定义                                            │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【自检与确认环节】Hoop 半自动化                            │
│    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                    │
│    │   自检模式   │ or │   教师审核   │ or │   专家确认   │                    │
│    │  (个人用户)  │    │  (教学场景)  │    │  (高端订阅)  │                    │
│    └─────────────┘    └─────────────┘    └─────────────┘                    │
│         ↓                    ↓                    ↓                          │
│    用户自己确认提示词    教师批注修改         专家审核反馈                      │
│    可设置自动通过规则    可退回重写           可设置审核节点                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【图像生成】GPT Image2                                    │
│    确认后的提示词 ──→ GPT Image2生成多方案（4-8张）                             │
│    • 支持风格参考图上传（Reference Image）                                     │
│    • 支持比例/尺寸预设（展板常用A1比例等）                                      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【出图画板】                                              │
│    多方案对比 ──→ 用户选择最合适的一张                                         │
│    ├─→ 放入画板（支持后续调整）                                               │
│    ├─→ 或上传自己的图片替代                                                   │
│    └─→ 不满意可返回提示词环节重新生成                                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【PPT工作流】                                             │
│    DeepSeek生成：                                                             │
│     • 汇报大纲（结构框架）                                                     │
│     • 每页内容要点                                                            │
│     • 演讲者备注（讲稿）                                                       │
│                                    ↓                                         │
│    Kimi生成PPT：                                                              │
│     • 接入Kimi PPT生成功能                                                    │
│     • 配套Skill（预设风格：学术/竞赛/汇报/简约）                               │
│     • 风格参考图/参考PPT上传                                                   │
│                                    ↓                                         │
│    PPT迭代：                                                                  │
│     • 手动修改（在线编辑）                                                     │
│     • AI辅助调整（"这页太满了，简化一下"）                                    │
│     • 版本对比与回溯                                                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【A1展板生成】GPT Image2                                  │
│    输入：                                                                     │
│     • DeepSeek生成的提示词大纲                                                │
│     • PPT效果图内容（已确认的设计成果）                                        │
│     • 风格/排版参考                                                           │
│    输出：                                                                     │
│     • A1尺寸展板（594mm × 841mm）                                             │
│     • 支持建筑/景观竞赛展板标准排版                                            │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                      【延迟确认与即时调整】                                    │
│    • 最终PPT和展板进入"待确认"状态                                           │
│    • 用户可延迟确认（异步审核）                                                │
│    • 确认前可随时调整任一节点参数，自动级联更新                                 │
│    • 确认后锁定版本，生成最终交付物                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、系统模块划分

### 2.1 前端层（轻量化Web）

| 模块 | 功能 |
|------|------|
| **项目工作台** | 新建/管理项目，项目分类筛选 |
| **节点画布** | 可视化工作流节点，可拖拽调整顺序/参数 |
| **提示词工坊** | 模板库、自定义编辑、历史版本、风格参考管理 |
| **图像画廊** | 多方案对比、画板编辑、上传图片、风格迁移 |
| **PPT编辑器** | 在线编辑、AI调整指令、版本历史 |
| **展板式预览** | A1尺寸预览、标注工具、输出设置 |
| **审核中心** | 自检/教师/专家三级确认流 |
| **订阅管理** | 套餐选择、用量统计、升级降级 |

### 2.2 后端层（API编排）

| 服务 | 职责 |
|------|------|
| **工作流引擎** | 节点DAG执行、状态机、参数级联更新 |
| **AI编排器** | 统一调度DeepSeek/GPT Image2/Kimi/Hoop |
| **提示词引擎** | 模板渲染、变量注入、优化建议 |
| **审核服务** | 确认流管理、权限控制、审核记录 |
| **文件服务** | 图片/PPT/展板存储、CDN、版本管理 |
| **订阅服务** | 支付、配额控制、用量计费 |

### 2.3 AI能力矩阵

| 能力 | 提供商 | 用途 |
|------|--------|------|
| 文本分析/生成 | DeepSeek | 需求分析、策略文本、讲稿、提示词大纲 |
| 图像生成 | GPT Image2 | 效果图、鸟瞰图、分析图、展板 |
| PPT生成 | Kimi | 汇报PPT一键生成 |
| 流程自动化 | Hoop | 节点衔接、条件触发、半自动化审批 |

## 四、技术栈建议

```
前端：Next.js 14 (App Router) + Tailwind + shadcn/ui
      ├─ 节点画布：ReactFlow
      ├─ PPT编辑：自研轻量编辑器 或 集成OnlyOffice
      └─ 图像预览：Fabric.js / Konva

后端：Node.js (NestJS) 或 Python (FastAPI)
      ├─ 工作流：Temporal.io 或自研状态机
      ├─ 数据库：PostgreSQL (项目数据) + Redis (队列/缓存)
      └─ 存储：阿里云OSS / AWS S3

AI接入：
      ├─ DeepSeek：官方API
      ├─ GPT Image2：Azure OpenAI / OpenAI API
      ├─ Kimi：Moonshot API (PPT生成)
      └─ Hoop：Hoop API 或 Webhook

部署：Vercel (前端) + 云服务器 (后端) / 或全栈Railway/Render
```

## 六、关键设计决策

### Q1：节点级联更新怎么实现？
**方案**：每个节点输出为不可变快照，下游节点订阅上游。当上游参数变更时，标记下游为"待刷新"，用户触发重跑或设置自动重跑规则。

### Q2：如何控制AI调用成本？
**方案**：
- 提示词模板预优化，减少Token消耗
- 图像生成采用"草稿→精修"两阶段（先用低价模型出草图）
- 订阅配额制，超限可购加油包

### Q3：教师/专家审核怎么落地？
**方案**：
- 生成分享链接（限时/密码保护）
- 审核人在独立页面批注
- 批注同步回工作流，用户可选择采纳/忽略

### Q4：PPT迭代怎么做到"可手动改，又可AI调"？
**方案**：
- Kimi生成初稿导出为可编辑格式
- 在线编辑器保留段落级AI指令（选中一段 → "简化这段话"）
- 修改历史用Operational Transform或Yjs协同算法

## 七、风险与应对

| 风险 | 应对 |
|------|------|
| AI生成图像质量不稳定 | 多方案生成+人工选择；提示词工程持续优化；允许用户上传参考图 |
| API成本不可控 | 分层订阅配额；草稿/精修两阶段；缓存复用 |
| Kimi PPT API限制 | 同时保留备用方案（如自研PPT模板引擎） |
| 用户学习成本高 | 内置教学项目；模板引导；渐进式功能开放 |
